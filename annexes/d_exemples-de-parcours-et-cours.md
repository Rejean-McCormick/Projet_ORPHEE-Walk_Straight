## 1. Résumé

Cette annexe présente des exemples de parcours et de cours pour le Projet Orphée: Walk Straight.

Le projet ne vise pas d’abord à transmettre des connaissances théoriques.  
Il vise à former des personnes capables de **produire des savoirs utiles** :

- plans;
- livres;
- modules de formation;
- présentations;
- guides;
- procédures;
- quiz;
- contenus multimédias;
- ressources multilingues;
- paquets offline;
- outils pédagogiques simples.

Le modèle repose sur l’apprentissage par production :

> **On apprend en fabriquant quelque chose qui peut servir à d’autres.**

Cette logique est cohérente avec le cycle kOA :

```text
Connaître → Choisir → Agir → Se souvenir → Mieux connaître
````

Le kOA Digital Ecosystem est justement défini comme une architecture qui transforme les contributions en connaissance, la connaissance en orientation, l’orientation en action coordonnée, et l’expérience en mémoire active. 

---

## 2. Publics visés

Les parcours peuvent être adaptés à plusieurs publics.

### 2.1 Pilote initial

Le premier pilote vise une cohorte hybride locale :

* étudiants du cégep;
* membres du personnel;
* acteurs communautaires;
* experts invités;
* partenaires régionaux;
* contributeurs volontaires;
* adultes en formation continue;
* personnes intéressées par la production de savoirs numériques.

### 2.2 Terrains spécialisés futurs

Après validation du pilote, les parcours peuvent être adaptés à :

* villages isolés;
* Basse-Côte-Nord;
* organismes communautaires;
* milieux à faible connectivité;
* personnes éloignées de la formation;
* milieux fermés;
* milieu carcéral, dans une étape ultérieure.

---

## 3. Principes pédagogiques

Les parcours doivent respecter les principes suivants.

### 3.1 Production avant théorie complète

Chaque cours doit mener à une production concrète.

Exemples :

* produire un plan;
* transformer un texte en module;
* créer une capsule;
* documenter une procédure;
* créer un support visuel;
* bâtir un glossaire;
* générer une version offline;
* préparer une ressource multilingue.

### 3.2 Progression courte

Chaque cours doit pouvoir être suivi en petites unités.

Format recommandé :

```text
Mini-leçon → Démonstration → Exercice guidé → Production → Révision → Portfolio
```

### 3.3 Accessibilité

Chaque ressource devrait pouvoir être adaptée en formats sobres :

* texte simple;
* PDF;
* HTML statique;
* diaporama exportable;
* paquet téléchargeable;
* clé USB;
* intranet;
* impression papier, si nécessaire.

### 3.4 IA encadrée

L’IA peut aider à produire, mais elle ne doit pas remplacer le jugement humain.

L’apprenant doit apprendre à :

* formuler une demande;
* vérifier une réponse;
* corriger;
* simplifier;
* citer;
* reformuler;
* adapter;
* documenter son processus.

### 3.5 Preuves de compétence

Chaque cours doit produire une preuve :

* fichier final;
* capture de processus;
* journal de production;
* grille de révision;
* version avant/après;
* commentaire réflexif;
* élément de portfolio.

La doctrine UCKK-Moodle prévoit déjà une logique de cours, compétences, preuves et badges, ce qui soutient ce type de structuration pédagogique.

# 4. Parcours A — Tronc commun Orphée

## ORP-101 — Introduction à Orphée: Walk Straight

**Objectif :** Comprendre le projet, sa mission, ses publics et ses règles de base.

### Compétences visées

* Expliquer la mission du projet.
* Distinguer formation, gouvernance, production et transfert.
* Comprendre la logique des milieux isolés et contraints.
* Identifier les règles éthiques de base.

### Productions

* fiche personnelle d’objectifs;
* carte des publics visés;
* résumé du projet en 150 mots;
* version simplifiée du projet pour un public externe.

### Exercices

1. Reformuler le projet pour un étudiant.
2. Reformuler le projet pour un partenaire communautaire.
3. Reformuler le projet pour un décideur institutionnel.
4. Identifier trois risques et trois bénéfices.

---

## ORP-102 — Littératie IA appliquée

**Objectif :** Apprendre à utiliser l’IA comme outil de production, sans lui déléguer le jugement.

### Compétences visées

* Rédiger un prompt clair.
* Vérifier une réponse IA.
* Identifier erreurs, hallucinations et imprécisions.
* Réviser un texte généré.
* Documenter l’usage de l’IA.

### Productions

* banque de prompts simples;
* fiche “avant/après”;
* guide d’usage responsable de l’IA;
* mini-rapport de vérification.

### Exercices

1. Demander à l’IA de résumer un texte.
2. Vérifier le résumé.
3. Corriger les erreurs.
4. Produire une version plus claire.
5. Documenter les limites de l’outil.

* fiche;
* glossaire;
* consigne;
* mini-guide.

## Niveau 2 — Production guidée

L’apprenant produit avec gabarits et procédures.

Productions typiques :

* module court;
* quiz;
* présentation;
* fiche structurée;
* contenu révisé.

## Niveau 3 — Production autonome

L’apprenant peut concevoir une ressource complète.

Productions typiques :

* module complet;
* mini-livre;
* paquet offline;
* capsule;
* guide partenaire.

## Niveau 4 — Coordination

L’apprenant peut aider d’autres personnes à produire.

Productions typiques :

* procédure;
* grille de révision;
* plan de production;
* tableau de suivi;
* rapport d’amélioration.

## Niveau 5 — Transfert

L’apprenant peut adapter un parcours à un nouveau contexte.

Productions typiques :

* adaptation territoriale;
* version multilingue;
* version milieu contraint;
* plan de déploiement;
* rapport de transfert.

# 17. Critères généraux d’évaluation

Chaque production peut être évaluée selon les critères suivants.

| Critère       | Description                                                     |
| ------------- | --------------------------------------------------------------- |
| Clarté        | Le contenu est compréhensible.                                  |
| Utilité       | Le contenu répond à un besoin réel.                             |
| Structure     | Le contenu suit un ordre logique.                               |
| Exactitude    | Les affirmations sont vérifiées.                                |
| Accessibilité | Le contenu peut être utilisé par plusieurs publics.             |
| Sobriété      | Le contenu ne dépend pas inutilement d’outils lourds.           |
| Multilingue   | Le contenu peut être adapté ou traduit.                         |
| Offline       | Le contenu peut être exporté ou utilisé hors ligne.             |
| Éthique       | Le processus respecte consentement, sécurité et non-coercition. |
| Portfolio     | La production peut servir de preuve de compétence.              |

# 20. Formule finale

Le modèle pédagogique d’Orphée: Walk Straight peut se résumer ainsi :

```text
Apprendre → Produire → Réviser → Gouverner → Archiver → Transférer
```

Formule courte :

> **Former des producteurs de savoirs, pas seulement des consommateurs de cours.**

Formule stratégique :

> **Chaque parcours doit créer une compétence, une preuve et une ressource réutilisable.**
