# Annexe C — Grilles d’évaluation

**Projet :** Orphée: Walk Straight  
**Chemin :** `orphee-walk-straight/annexes/c_grilles-evaluation.md`  
**Statut :** Annexe opérationnelle  
**Version :** 0.1  
**Rôle :** Fournir des grilles simples pour évaluer les apprentissages, les productions, les portfolios, les terrains de transfert et la maturité CCTT du projet.

---

## 1. Fonction de l’annexe

Cette annexe fournit des grilles d’évaluation pour le projet **Orphée: Walk Straight**.

Ces grilles servent à :

- suivre la progression des participants;
- évaluer les productions concrètes;
- documenter les compétences acquises;
- soutenir les portfolios;
- encadrer l’usage de l’IA;
- vérifier la qualité des contenus;
- préparer le transfert vers des milieux isolés ou contraints;
- produire des preuves utiles pour une trajectoire CCTT.

Les grilles ne sont pas conçues comme des examens punitifs.

Elles servent à rendre visibles :

- ce qui est compris;
- ce qui est produit;
- ce qui doit être amélioré;
- ce qui peut être reconnu;
- ce qui peut être transféré.

---

## 2. Échelle commune d’évaluation

Toutes les grilles utilisent une échelle simple à quatre niveaux.

| Niveau | Nom | Description |
|---|---|---|
| 0 | Non démontré | La compétence ou le critère n’est pas observable. |
| 1 | Débutant | La compétence apparaît, mais avec aide importante ou erreurs fréquentes. |
| 2 | Fonctionnel | La compétence est démontrée de manière utilisable, avec corrections mineures. |
| 3 | Solide | La compétence est maîtrisée, transférable et documentée. |
| 4 | Exemplaire | La compétence dépasse les attentes et peut servir de modèle à d’autres. |

### Règle d’interprétation

```text
0 = absent
1 = en apprentissage
2 = utilisable
3 = transférable
4 = exemplaire
````

Le niveau **2** est le seuil minimal pour considérer une compétence comme démontrée.

Le niveau **3** est le seuil recommandé pour l’inclure dans un portfolio officiel.

Le niveau **4** indique qu’une production peut être utilisée comme exemple pédagogique.

### Interdictions absolues

L’usage de l’IA est refusé si la production contient :

* fraude;
* usurpation;
* manipulation;
* hameçonnage;
* données sensibles non autorisées;
* contenu haineux;
* contenu violent opérationnel;
* cybersécurité offensive;
* contournement de règles institutionnelles;
* production faussement attribuée à une personne.

## 21. Règles de conservation des évaluations

Les évaluations doivent être conservées avec prudence.

Principes :

* minimiser les données personnelles;
* ne conserver que ce qui est utile;
* séparer les commentaires pédagogiques des informations sensibles;
* anonymiser lorsque possible;
* obtenir les consentements nécessaires;
* éviter les jugements humiliants;
* documenter les productions plutôt que catégoriser les personnes;
* permettre la correction;
* permettre le retrait lorsque le cadre le prévoit.

## 23. Phrase maîtresse

> Une compétence n’est pas seulement ce qu’une personne sait; c’est ce qu’elle peut produire, expliquer, corriger, documenter et transférer dans un contexte réel.
